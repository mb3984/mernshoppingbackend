🍎🍅 MERN Fruits & Vegetables App – Backend Explained

This backend is REST API based, built with:

Node.js

Express

MongoDB + Mongoose

JWT Authentication

Role-based access (Admin / User)

1️⃣ Project Purpose (What problem it solves)

This backend is used for an online fruits & vegetables store where:

Admin

Adds fruits & vegetables

Updates prices & stock

Deletes products

Manages users

Users

Register & login

View products

Add products to cart

Place orders

👉 Same logic as restaurant app, just items are fruits & vegetables.

2️⃣ Backend Folder Responsibility
🔹 server.js / index.js

This is the entry point of backend.

Responsibilities:

Load environment variables

Connect MongoDB

Start Express server

Enable JSON & CORS

Register routes

📌 Without this file → backend won’t start.

3️⃣ Database Connection (MongoDB)
🔹 config/db.js

What it does:

Connects backend to MongoDB Atlas / Local MongoDB

Uses MONGO_URI from .env

Why important?

All users, products, orders are stored here

If DB fails → app stops (safe behavior)

4️⃣ Authentication System (JWT)
🔹 Why Authentication?

Because:

Users must login to place orders

Admin must be protected from normal users

🔹 JWT Flow (Simple)

User logs in

Backend creates JWT token

Token is sent to frontend

Frontend sends token in Authorization header

Backend verifies token for protected routes

5️⃣ Middleware (Security Layer)
🔹 authMiddleware.js

This is the gatekeeper 🚪

✅ protect middleware

Checks:

Token exists or not

Token is valid or not

Extracts user data from token

Attaches user info to req.user

📌 Used for:

Cart

Orders

Admin routes

✅ admin middleware

Checks:

req.user.role === "admin"

Allows only admins

📌 Used for:

Add product

Update product

Delete product

View all users

6️⃣ User Module (Normal User + Admin)
🔹 User Features
👤 User can:

Register

Login

View products

Add to cart

Place order

👑 Admin can:

View all users

Promote user to admin

🔹 User Routes

/api/users

Route	Purpose
POST /register	Create new user
POST /login	Login & get token
GET /	Get all users (Admin only)
PUT /:id/make-admin	Make user admin
🔹 Role Concept

Each user has:

role: "user" (default)

role: "admin"

This enables role-based access control.

7️⃣ Product Module (Fruits & Vegetables)
🔹 Product Features

Each product contains:

Name

Description

Image

Category (Fruit / Vegetable)

Price per Kg

Stock

Seasonal or not

🔹 Product Routes

/api/products

Route	Purpose	Access
GET /	View all products	Public
GET /:id	View single product	Public
POST /	Add product	Admin
PUT /:id	Update product	Admin
DELETE /:id	Delete product	Admin
🔹 Why Admin-only Product Control?

Because:

Prices & stock must be controlled

Prevent fake products

Real-world ecommerce rule

8️⃣ Cart Feature (Logical Flow)
🛒 Cart Concept

Each user has a cart

Cart stores:

Product ID

Quantity

Price snapshot

🔁 Flow:

User logs in

Adds product to cart

Backend checks stock

Cart updates quantity

📌 Cart is temporary until order placed.

9️⃣ Order Feature (Checkout)
📦 Order Flow

User clicks Place Order

Backend:

Validates cart

Calculates total price

Reduces product stock

Order stored in DB

Cart cleared

🧾 Order Data Includes:

User ID

Products list

Quantity

Total amount

Order status (Pending / Delivered)

🔟 Security Features

✅ JWT authentication
✅ Admin-only access
✅ Password hashing (bcrypt)
✅ Protected routes
✅ Centralized error handling
